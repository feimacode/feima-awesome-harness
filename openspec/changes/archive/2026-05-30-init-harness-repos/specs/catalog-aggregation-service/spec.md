## ADDED Requirements

### Requirement: Catalog service directory structure

The catalog repo SHALL contain a `catalogs/` directory with per-repo subdirectories following the pattern `catalogs/<repo-name>/catalog.json`.

#### Scenario: Official catalog directory
- **WHEN** the catalog service is initialized
- **THEN** `catalogs/feima-awesome-harness/` directory SHALL exist
- **AND** it SHALL contain `catalog.json` describing feima-awesome-harness content

#### Scenario: Community catalog directory
- **WHEN** the catalog service is initialized
- **THEN** `catalogs/community/` directory SHALL exist
- **AND** it SHALL be empty initially (for future contributor entries)

### Requirement: Catalog.json schema

Each `catalog.json` SHALL contain metadata describing skills, agents, and flows from a single source repo.

#### Scenario: Catalog.json structure
- **WHEN** a catalog.json file is created
- **THEN** it SHALL contain `flows`, `skills`, `prompts` arrays
- **AND** each entry SHALL have `id`, `name`, `description`, `source`, `tags`, and `type` fields
- **AND** flow entries SHALL include `orchestration`, `roles`, and `uses_skills` fields

#### Scenario: Source field format
- **WHEN** a catalog entry specifies `source`
- **THEN** it SHALL use one of: `github:<owner>/<repo>`, `gist:<id>`, or `pkg:<marketplace>/<plugin>` format

### Requirement: Index.json auto-generation

The catalog repo SHALL contain `index.json` at root, auto-generated by merging all catalog.json files from `catalogs/` directory.

#### Scenario: Index.json structure
- **WHEN** build-index.js runs
- **THEN** `index.json` SHALL be created at repo root
- **AND** it SHALL contain merged entries from all catalog.json files
- **AND** each entry SHALL include `provider` derived from directory path

### Requirement: Build-index CI workflow

The catalog repo SHALL contain a GitHub Actions workflow that runs `build-index.js` on catalog changes.

#### Scenario: Workflow triggers
- **WHEN** files in `catalogs/` directory are modified
- **THEN** `.github/workflows/build-index.yml` SHALL run
- **AND** it SHALL execute `scripts/build-index.js`
- **AND** it SHALL commit updated `index.json` to the repo

### Requirement: Build-index script

The catalog repo SHALL contain `scripts/build-index.js` that merges catalog.json files and writes index.json.

#### Scenario: Script functionality
- **WHEN** build-index.js is executed
- **THEN** it SHALL scan all catalog.json files in `catalogs/` directory
- **AND** it SHALL merge them into a single index.json
- **AND** it SHALL derive `provider` from directory name (e.g., `catalogs/feima-awesome-harness/` → `provider: "feima-awesome-harness"`)

#### Scenario: Manifest fallback mode
- **WHEN** a catalog.json entry has `source: "github:<owner>/<repo>"` but no explicit skill/flow entries
- **THEN** build-index.js MAY fetch plugin manifests from that repo
- **AND** it MAY derive catalog entries from marketplace.json or plugin.json content
